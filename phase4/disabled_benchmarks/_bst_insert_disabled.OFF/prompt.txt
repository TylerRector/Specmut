Write a Lean 4 specification for inserting a key into a binary search tree
modeled as a sorted list of natural numbers.

Requirements:
- Declare `bstInsert : Nat → List Nat → List Nat` as an `axiom`.
- You may also declare `bstMax : List Nat → Nat` as an `axiom` if useful.
- Define helper predicates as `def` with concrete recursive bodies.
- State theorems (using `:= sorry`) that fully specify the BST invariant.

The specification should be self-contained (no imports beyond Lean's core).

Properties that a strong specification should capture:
- The inserted key is a member of the resulting structure.
- The BST ordering invariant is preserved under insertion.
- Insertion increases the size by one (or by zero if the key is already present).

Output ONLY the Lean code.  Do not include explanations or commentary.
