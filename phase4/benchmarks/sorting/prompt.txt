Write a Lean 4 specification for a sorting function over lists of natural numbers.

Requirements:
- Declare `sort : List Nat → List Nat` as an `axiom` (no implementation needed).
- Define helper predicates as `def` with concrete recursive bodies.
  Do not use `axiom` for predicates — they must have computable definitions.
- State theorems (using `:= sorry`) that together fully specify correct sorting.

The specification should be self-contained (no imports beyond Lean's core).

Properties that a strong specification should capture:
- The output is sorted in non-decreasing order.
- The output has the same length as the input.
- The output is a permutation of the input (same multiset of elements).

Output ONLY the Lean code.  Do not include explanations or commentary.
